Week 2 - Events and States

The goal of this week is to listen to events like tapping, scrolling, and editing and do things like change the position of views, navigate between screens, and transition to or from loading states.

By the end of the week, you should be able to take a set of mocks, and implement them with a high degree of fidelity and interactability, including error and loading states. The homework this week is to do that for a set of Carousel mocks.

Demos

  

Skills Overview

  • Events and Delegates
    • Events via targets and selectors
    • Delegation for simple controls like UIAlertView and UIActionSheet
    • Handling UITextField delegate events
    • Handling scroll view events
  • Application states
    • Mocking an transient loading state
    • Transitioning between empty and populated states
    • Performing selectors after delays

Refer to the Week 2 Skills checklist to review the full topics introduced this week.

Challenge #2: Furry Friends

Assets

You can download the Facebook assets here.

Milestones

  1. Setup: Create a new project, disable Auto Layout, and import assets from above. New Project
  2. Make a screen to move the Rabbit around Creating Custom View Controllers
    • Create a RabbitViewController.swift file and associate it with a view controller in the Storyboard.
    • Add the rabbit image with 4 buttons for left, right, up, and down. Configure a Button
    • Add a button to hide the rabbit.
    • Add a button to show the rabbit again.
  3. Feeding the rabbit
    • Add a leaf button to feed the rabbit and make it get bigger. Using View Transforms
    • Keep track of the current scale of the rabbit in a property. Add the property at the top of the file, right under the class name.
      • var scale: CGFloat! = 1
    • Tapping the leaf button should increase the scale.
  4. Shrinking the rabbit
    • Add a tear button that represents when the rabbit gets sad and makes it shrink. Using View Transforms
    • Use the same scale variable you made in the previous step to keep track of the scale as the rabbit shrinks.
    • Tapping the tear button should decrease the scale.
  5. Add animation to the scale, so the rabbit grows or shrinks smoothly. Animating View Properties
  6. Make the rabbit go to where you tapped. Using Gesture Recognizers
  7. Resetting the rabbit
    • Add a button to reset the rabbit. Using View Transforms
  8. Rotating the rabbit
    • Add buttons to rotate the rabbit. Using View Transforms
  9. Scroll Views
    • Add parallax between the rabbit and the field. Using UIScrollView
    • Add welcome screens with UIPageControl. Using UIPageControl
  10. Chatting with the rabbit
    • Add a textfield, a button, and a label to be able to chat with the rabbit. After tapping the button, you should read the textfield and update the label with the rabbit's response. Just support saying "Hello" for now.
    • Add a generic response if the text in the textfield isn't recognized.
    • Use case: Add the ability to respond to "What's your name?" and "Will you be my friend?"
    • Use case: Add a delay before the label is updated.
    • Polish: Clear the label after tapping the button while you're waiting for the rabbit to respond.
    • Add a UIActivityIndicatorView. Call startAnimating when the button is tapped and stopAnimating after the delay. Using UIActivityIndicatorView
  11. Login Screen
    • Create a login screen. Add elements for the email field, password field, and login button. Configure the password field for secure entry.
    • Create an action on login button. Call performSegueWithIdentifier on the login button. Using Modal Transitions
    • Add a delay after clicking the login button. Add a UIActivityIndicatorView. Calling Method After Delay, Using UIActivityIndicatorView
    • Check the email and password. If it's correct, segue. If it's incorrect, display a UIAlertView. Using UIAlertView
    • Disable the login button if there is no text in either the login field or the password field.
    • Add a logout button to the chat view. Tapping the logout button dismisses the view controller.
    • Tapping the logout button displays an action sheet confirming that you want to logout.

Week 2 Project: Carousel

Due: Monday, September 28th at 11:59pm

Overview

The purpose of this homework is to leverage views, view properties, and events to create a high fidelity prototype that is difficult to distinguish from a production app. We're going to use the techniques from this week to implement the Carousel app from the signed out state to the basic signed in state.

Project Requirements

  • Static photo tiles on the initial screen
    • Optional: Photo tiles move with scrolling
  • Sign In
    • Tapping on email/password reveals the keyboard and shifts the scrollview and Sign In button up.
    • User sees an error alert when no email is present or no password is present.
    • User sees a loading screen upon tapping the Sign In button.
    • alertController.dismissViewControllerAnimated(true, completion: nil) dismisses the alert controller loading screen with no buttons.
    • User sees an error alert when entering the wrong email/password combination.
    • User is taken to the tutorial screens upon entering the correct email/password combination.
    • Optional: When the keyboard is visible, if the user pulls down on the scrollview, it will dismiss the keyboard.
    • Optional: On appear, scale the form up and fade it in.
  • Optional: Sign Up
    • Optional: Tapping in the form reveals the keyboard and shifts the scrollview and "Create a Dropbox" button up.
    • Optional: Tapping the Agree to Terms checkbox selects the checkbox.
    • Optional: Tapping on Terms shows a webview with the terms.
    • Optional: User is taken to the tutorial screens upon tapping the "Create a Dropbox" button.
  • Tutorial Screens
    • User can page between the screens
    • Optional: User can page between the screens with updated dots
    • Optional: Upon reaching the 4th page, hide the dots and show the "Take Carousel for a Spin" button.
  • Image Timeline
    • Display a scrollable view of images.
    • User can tap on the conversations button to see the conversations screen (push).
    • User can tap on the profile image to see the settings view (modal from below).
  • Settings
    • User can dismiss the settings screen.
    • User can log out
  • Optional: Learn more about Carousel
    • Optional: Show the "Learn more about Carousel" button in the photo timeline.
    • Optional: Tap the X to dismiss the banner
    • Optional: Track the 3 events:
      • View a photo full screen
      • Swipe left and right
      • Share a photo
    • Optional: Upon completion of the events, mark them green.
    • Optional: When all events are completed, dismiss the banner.

Milestones

The key to implementing a complex app is to break it up into a bunch of small pieces until it's easy to implement any given piece. You should try implementing this homework by following the milestones below. Be sure to run the simulator frequently as you hit milestones.

  1. Setup:
    • Create a new project (disable Auto Layout). New Project
    • Add the image assets above. Adding Image Assets
    • Configure the app icon and splash screen. App icon and launch image
    • Add the Common.swift file to your project. Be sure to select "Copy items if needed".
  2. Intro Screen
    • In the Storyboard, add a custom view controller for the IntroViewController. Creating Custom View Controller
    • Configure the view controller to be freeform, so that you can make it long enough to see the entire intro page. Creating Free Form View Controller
    • Add UIImageViews for each photo tile in the region below. Using UIImageView
    • Configure the content size of the scroll view. Using UIScrollView
    • Optional: Register for the scroll view scroll events Registering for scroll events
    • Optional: Whenever a scroll event happens, configure the transform property on the image view. Using View Transforms,Animating View Properties
  3. Sign In
    • Add custom nav bar as a UIImageView, form background, text fields, text, and Sign In button.
    • Tapping the Sign In button with either a blank email or blank password should show an error alert. Using UIAlertController
    • Tapping the Sign In button with a populated email and password field should display a loading alert for 2 seconds. After the 2 seconds are up, you should do one of the following (Calling a Method After Delay):
    • Display an error for incorrect email/password.
    • Go to the tutorial screens.
  4. Tutorial Screens
    • Create a custom free form view controller that is wide enough for 4 screens. Creating Custom View Controllers, Creating a Free Form View Controller
    • Add a UIScrollView with paging enabled and 4 UIImageViews for the welcome screens (Note: we're explicitly not using UIPageViewController, which requires extra coding, to build these screens)
    • Add a UIPageControl (should be outside the scrollview) and link it with the scrollview. Using UIPageControl
    • Add the "Take Carousel for a Spin" button (should be outside the scrollview). Should have an initial alpha of 0, set the alpha to 1 upon reaching the 4th page.
    • Tapping the "Take Carousel for a Spin" button should launch the Image Timeline (in a Navigation Controller) modally.
  5. Image Timeline
    • Add a UIImageView for the custom nav bar and a scrollview for the image feed.
    • Add buttons for Settings and Conversations.
    • Tapping the Conversation button should push the ConversationsViewController.
    • Tapping the Settings button should modally present the SettingsViewController.
  6. Create Account
    • Add custom nav bar, form background, text fields, text, terms text and button, and Create Account button.
    • Upon showing the keyboard, move the elements to be visible. Registering for Keyboard Events
    • Add a selected state for the terms button. Configure a Button
    • Terms opens webview w/ mobile terms of service page.

Week 2 Group Milestone - Wireframing

The group project has weekly milestones that reinforces the lessons from the previous week. The group project is an important validation that you can apply the concepts from the homework to an arbitrary project. Also, it's an opportunity to be creative an have a little fun with iOS.

This week's milestone is to decide on an app idea and create basic wireframes. Treat this as a time-constrained hackathon and just go with the best idea you can think of in an initial brainstorming session.

Here are some possible directions:

  • Incorporate your current or upcoming project work. The advantage to this is you can log more hours on iOS in the next 6 weeks without feeling as guilty.
  • Clone an app, but add either interaction or product twists. That way, you don't have to design a product from scratch.
  • Something completely different. This can be the most rewarding, although it's the most time consuming.

Your milestone for next week will be to create interactive mocks in Xcode. The wireframes this week can be low or medium fidelity, but they have to be a complete product/UX story.

Submission instructions

Create a repository on Github with a project codename. Check in your wireframes and add timothy1ee as a collaborator in the project. The repository can be public or private.
